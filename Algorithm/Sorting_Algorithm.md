# 정렬 알고리즘
## 1 기본 개념
### 1.1 공간 복잡도(Space Complexity)
- **제자리 정렬(In-Place)**  
  추가 메모리 없이 가능한 정렬이 가능한 것을 말한다.
- **비제자리 정렬(Not In-Place)**  
  정렬을 위해 추가 메모리 공간이 필요한 것을 말한다.

### 1.2 안정성(Stability)
> 같은 키 값을 가진 요소들의 원래 순서가 유지되는가?

**안정적 정렬** → 중복 요소의 순서가 유지됨  
**불안정 정렬** → 중복 요소의 순서가 바뀔 수 있음  

**안정성이 중요해지는 경우** → 다중 기준 정렬 시

## 2 알고리즘
### 2.1 Bubble Sort
| 정렬                 | 평균 시간복잡도     | 최악 시간복잡도     | 안정성          | In-place          |
| ------------------ | ------------ | ------------ | ------------ | ----------------- |
| **Bubble Sort**    | O(N²)        | O(N²)        | **Stable**   | 가능 ✔            |

배열의 처음부터 인접한 두 원소 비교  
→ 순서가 잘못된 경우 swap  
→ 이를 (n-1)번 반복

### 2.2 Selection Sort

| 정렬                 | 평균 시간복잡도     | 최악 시간복잡도     | 안정성          | In-place          |
| ------------------ | ------------ | ------------ | ------------ | ----------------- |
| **Selection Sort** | O(N²)        | O(N²)        | **Unstable** | 가능 ✔            |

매 단계에서 가장 작은 원소(오름차순)를 선택해 앞쪽에 배치하는 정렬  
(항상 `O(n²)`)

### 2.3 Insertion Sort

| 정렬                 | 평균 시간복잡도     | 최악 시간복잡도     | 안정성          | In-place          |
| ------------------ | ------------ | ------------ | ------------ | ----------------- |
| **Insertion Sort** | O(N²)        | O(N²)        | **Stable**   | 가능 ✔            |

이미 정렬된 부분에 새로운 원소를 적절한 위치에 삽입하는 방식  
→ 정렬된 부분을 유지하며 확장됨  
(거의 정렬된 배열에서는 가장 빠름)  

두 번째 원소부터 이전 원소들과 비교하면서 적절한 위치에 삽입.

### 2.4 Merge Sort

| 정렬                 | 평균 시간복잡도     | 최악 시간복잡도     | 안정성          | In-place          |
| ------------------ | ------------ | ------------ | ------------ | ----------------- |
| **Merge Sort**     | O(N log N)   | O(N log N)   | **Stable**   | X (보조 배열 필요) |

**분할 정복(Divide & Conquer)** 기반  
→ 배열을 반으로 나누고 정렬 후 병합  
→ 재귀적으로 수행  

병합할 때는 이미 각각이 정렬된 상태  
→ 한 칸 씩 밀면서 선택하면 됨.

### 2.5 Quick Sort

| 정렬                 | 평균 시간복잡도     | 최악 시간복잡도     | 안정성          | In-place          |
| ------------------ | ------------ | ------------ | ------------ | ----------------- |
| **Quick Sort**     | O(N log N)   | O(N²)        | **Unstable** | 가능 ✔            |

피벗(pivot)을 기준으로 작은 값과 큰 값을 양쪽으로 분할 후 재귀 정렬  
→ 재귀적으로 수행  
→ 평균적으로 매우 빠르지만, 편향되면 `O(n²)`

### 2.6 Heap Sort

| 정렬                 | 평균 시간복잡도     | 최악 시간복잡도     | 안정성          | In-place          |
| ------------------ | ------------ | ------------ | ------------ | ----------------- |
| **Heap Sort**      | O(N log N)   | O(N log N)   | **Unstable** | 가능 ✔            |

**힙(Heap)** 자료구조(완전 이진 트리) 기반으로 정렬 수행  

오름차순 → Max Heap을 사용해서 뒤에서부터 채움  
내림차순 → Min Heap을 사용해서 뒤에서부터 채움
